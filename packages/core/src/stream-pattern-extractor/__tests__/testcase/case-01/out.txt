## 总结

本次生成了三个功能各异的卡片 Schema，展示了生成式 UI 在不同场景下的应用：

### 1. **表单交互卡片**
- **组件**：使用 `TinyForm`、`TinyFormItem`、`TinyInput`、`TinyButton` 构建
- **特性**：实现了完整的表单双向绑定功能，通过 `modelValue` 的 `model: true` 属性确保数据同步
- **交互**：集成 `saveState` 和 `continueChat` Action，实现数据持久化和对话连续性
- **适用场景**：用户信息收集、反馈提交、数据录入等

### 2. **数据展示卡片**
- **组件**：结合 `TinyChartPie`、`TinyChartLine`、`TinyGrid` 和布局组件
- **特性**：使用栅格系统 (`TinyLayout`/`TinyRow`/`TinyCol`) 实现响应式布局
- **数据可视化**：饼图展示分类占比，折线图展示趋势变化，表格展示明细数据
- **适用场景**：数据仪表盘、项目报告、统计概览等

### 3. **内容导航卡片**
- **组件**：以 `TinyTabs` 为核心，配合 `Icon`、`div`、列表组件
- **特性**：标签页切换与状态管理，通过 `onChange` 事件保存当前状态
- **内容组织**：分层展示产品、解决方案、技术支持等信息
- **适用场景**：产品介绍、服务说明、知识库导航等

### 设计要点总结：
1. **遵循 Schema 规范**：所有卡片均以 `Page` 为根节点，`TinyCard` 包裹内容
2. **状态管理**：合理使用 `state` 和 `methods`，确保交互功能完整
3. **组件组合**：充分利用提供的组件库，构建丰富多样的 UI 界面
4. **交互集成**：通过 `callAction` 与系统 Action 集成，实现业务逻辑
5. **布局美观**：使用栅格系统和适当的间距，确保视觉层次清晰

这些卡片 Schema 可以直接在支持生成式 UI 的平台上渲染使用，展示了从简单表单到复杂数据展示的完整能力。
